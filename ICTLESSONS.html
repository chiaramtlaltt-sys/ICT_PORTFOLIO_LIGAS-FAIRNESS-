JAVASCRIPT

    Purpose

  • Creates dynamic, interactive user experiences
  • - Handles UI rendering, content loading, real-time updates, personalized feeds
  • Controls video features (captions, volume)
  • Tracks user activity (e.g., Meta Pixel)

- Static web page: Delivered exactly as stored (e.g., photographer’s portfolio)

- Dynamic website: Displays different content based on user actions/preferences (e.g., social media platforms)

CONCEPTS

    HISTORY

  • - Developed by Brendan Eich at Netscape (originally Mocha, later LiveScript)
  • - Trademark of Oracle Corporation, licensed to Mozilla Foundation
  • Client-side, high-level, interpreted, object-oriented scripting language

    What can Javascript do?

  • - Enhances webpages with interactivity and dynamic content
  • Pag angat, pagsulong, o paglago ng pangkalahatang aspeto ng buhay. (Bon at Bon)
  • Creates and manages cookies (stores user info/preferences)
  • - Validates form data (saves server processing time)

    Features

  • - Combines with CSS for DHTML (effects like scrollers, trails, falling snow)
  • - Detects user actions beyond HTML’s capabilities
  • Dynamic typing (variable type can change at runtime)
  • Runs locally for faster interaction

    Characterstics

  • License-free
  • - Quick to download effects

    Disadvantages

  • -Not supported on some devices/browsers (e.g., older PDAs, some mobiles)
  • -- Can be disabled for security reasons
  • - Exposed source code (can be reverse engineered)
  • - Potential for malicious scripts via DOM

Where to place javascript?

    Head Tag

  • - Scripts here are pre-loaded before events are triggered.
  • - - Ideal for function definitions or code that must be ready before the page loads.

    Body Tag

  • - Scripts execute as the page loads.
  • - Often used for inline actions or immediate content output.

    Head and Body Tag

  • - Scripts can be placed in both sections.
  • - Useful when you need pre-loaded functions and also inline execution.

    External Javascript

  • - Save code in a .js file and link it using the src attribute.
  • - Keeps code reusable and cleaner.

Characteristics of Javascript

    Interpreted Language

  • -Runs inside a browser, not compiled beforehand.
  • Source code is visible via “View Source.”

    Scripting Language

  • - Designed for simple, repetitive tasks.
  • - Provides instructions to other programs (e.g., HTML/CSS).

    Client-Side Scripting

  • Executed on the user’s machine (not the server).
  • Faster interaction compared to server-side scripts.

    Object-Oriented Programming

  • - Treats elements as objects with properties and methods.
  • - Methods = actions objects can perform.
  • Properties = attributes of objects.

    Concepts to remember:

  • - Objects: Browser items (window, document, status bar, date/time).
  • - Methods: Functions tied to objects
  • - Properties: Subsections of objects
  • - Events: User/browser actions (click, hover, load).
  • - - Parameters: Data passed to functions/methods.
  • -- Types: Variable categories (string, integer, boolean, etc.).
  • - - Type Conversion: Changing variable type (e.g., string → integer).

    Objects, Properties, and Methods

  • - - Window Object
  • -- Document Object
  • - Math Object
  • - Navigator Object

    Variables & Data Type

  • - - Variable: Symbolic container for values.
  • - - Data Types:
  • - - Declaring Variables: - var firstname; - var firstname = "Crispin";
  • -- Naming Rules: - Letters, digits, underscore allowed. - No spaces or punctuation. - Cannot start with a digit. - Case-sensitive (firstname ≠ Firstname).

What are Events?

    Events:

  • Actions triggered by user interaction or browser activity (mouse clicks, keyboard strokes, image loading, form submission).
  • Usually tied to functions; functions need events to execute.

    Common Event Triggers:

  • -triggered when entering or leaving a webpage (e.g., check browser version, manage cookies).
  • - Used for form field validation.
  • - Validates all form fields before submission
  • - Used for animations
  • - Triggered by clicking elements

    Popup Boxes

  • - Alert Box → Displays a message
  • - Prompt Box → Requests user input (returns value or null).
  • - - Confirm Box → Asks user to verify/accept (returns true or false).

    Variables

  • - Temporary storage for values/expressions
  • - case sensitive, must begin with a letter or underscore
  • - adapt to assigned values

    Operators

  • - - Arithmetic Operators
  • - - Concatenation: + can join strings (e.g., "Good" + "Morning" → "Good Morning").
  • - - Order of Precedence: Parentheses → Multiplication/Division/Modulus → Addition/Subtraction.
  • - Increment/Decrement (++, --): - x++ vs ++x → different results when used inside expressions.

    Comparison and Logical statements

  • - Comparison Operators: used to evaluate equality/Difference
  • - Logical Operators: Combine conditions

    Conditional Statements

  • - if → Executes code if condition is true.
  • - if-else → Executes one block if true, another if false.
  • - if-else if-else → Multiple conditions checked sequentially.
  • - switch → Selects one block of code based on a single variable’s value.
